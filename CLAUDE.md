# Void Survivors — CLAUDE.md

## 프로젝트 개요
도형 기반 2D 뱀서라이크 브라우저 게임.
차원의 균열에 빨려 들어간 존재가 파편을 모아 살아남는 게임.

## 기술 스택
- Phaser.js 3 (게임 엔진)
- HTML5 Canvas
- Vanilla JavaScript
- Supabase (로깅 전용)
- GitHub Pages (배포)

## 프로젝트 구조
void-survivors/
├── index.html
├── src/
│   ├── scenes/        → 게임 씬 (Boot, Game, UI, GameOver)
│   ├── entities/      → 플레이어, 적, 보스, 아이템
│   ├── systems/       → 웨이브, 아이템 조합, 레벨업
│   └── config/        → 게임 설정, 밸런스 수치
├── assets/            → 도형 에셋 (MVP는 Phaser 도형 사용)
└── CLAUDE.md

## 핵심 게임 루프
1. 준비 페이즈 (30초) → 맵에 흩어진 파편 아이템 줍기
2. 전투 페이즈 → 웨이브 클리어 + 보스
3. 라운드 클리어 → 아이템 강화 or 새 아이템 선택
4. 같은 계열 아이템 7개 → 극강 아이템 변환

## 코딩 규칙
- 모든 주석은 한국어로
- 파일 하나 최대 200줄
- 씬 간 데이터는 Phaser Registry 사용
- 밸런스 수치는 반드시 src/config/ 에서 관리
- 절대 하드코딩 금지

## 에이전트 공통 규칙
- 코드 수정 전 반드시 현재 구조 파악
- 기능 추가 시 config/ 수치 먼저 정의
- 버그 수정 시 원인 한 줄 주석으로 남기기
- MVP 범위 벗어나는 기능 제안 금지

## 에이전트별 모델 배정

| 에이전트 | 모델 | 용도 |
|---|---|---|
| Planner | claude-opus-4-6 | 기능 설계, 구조 결정, 복잡한 판단 |
| Coder | claude-sonnet-4-6 | 코드 구현, 버그 수정, 리팩토링 |
| Reviewer | claude-sonnet-4-6 | 코드 리뷰, 버그 탐지, 성능 검토 |
| Artist | claude-haiku-4-5 | 색상/도형 규칙 적용, 에셋 규격 확인 |
| Tester | claude-haiku-4-5 | 밸런스 체크, 버그 재현, 반복 테스트 |

### 모델 선택 기준
- **Opus** → 판단과 설계가 필요한 작업 (느리지만 정확)
- **Sonnet** → 구현과 분석이 필요한 작업 (속도/품질 균형)
- **Haiku** → 반복적이고 단순한 작업 (빠르고 저렴)

### 주의사항
- Planner는 Opus를 쓰되 컨텍스트를 짧게 유지
- Coder는 한 번에 하나의 파일만 작업
- Haiku는 판단이 필요한 작업에 쓰지 않는다

## 토큰 최적화 규칙
- 작업 지시서에 명시된 파일만 읽기
- 읽지 않아도 되는 파일: 작업 범위 밖 파일
- 이미 읽은 파일 재읽기 금지
- config/ 파일은 수치 추가할 때만 열기
