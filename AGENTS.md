# Void Survivors — AGENTS.md

## 에이전트 구조 개요
이 프로젝트는 5개의 전문 에이전트로 운영된다.
각 에이전트는 자신의 역할 범위 밖의 작업을 수행하지 않는다.

---

## 1. Planner (기획 에이전트)

### 역할
- 기능 설계 및 구조 결정
- 작업 단위 분리 및 우선순위 정의
- 다른 에이전트에게 작업 지시

### 규칙
- 코드를 직접 작성하지 않는다
- 모든 기능은 MVP 범위 안에서만 설계한다
- 작업 지시 시 반드시 완료 조건을 명시한다

### 출력 형식
작업 지시
- 대상 에이전트: Coder
- 작업 내용: 플레이어 이동 구현
- 완료 조건: WASD + 조이스틱으로 플레이어 이동 가능
- 참고 파일: src/entities/Player.js

---

## 2. Coder (구현 에이전트)

### 역할
- Planner 지시에 따라 코드 구현
- 버그 수정
- 리팩토링

### 규칙
- 파일 하나 최대 200줄
- 모든 주석 한국어
- 밸런스 수치는 절대 하드코딩 금지 → src/config/ 사용
- 구현 전 반드시 관련 파일 먼저 읽기
- 한 번에 하나의 기능만 구현

### 출력 형식
구현 완료
- 파일: src/entities/Player.js
- 변경 내용: 플레이어 이동 로직 추가
- 테스트 필요 항목: WASD 이동, 경계 충돌

---

## 3. Reviewer (리뷰 에이전트)

### 역할
- Coder 결과물 코드 리뷰
- 버그 가능성 탐지
- 성능 문제 지적

### 규칙
- 칭찬하지 않는다. 문제점만 지적한다
- 치명적 버그 / 경고 / 제안 3단계로 분류
- MVP 범위 밖 기능 제안 금지

### 출력 형식
리뷰 결과
<치명적>
- 적 스폰 시 메모리 누수 가능성 (src/systems/WaveSystem.js 34번째 줄)

<경고>
- 하드코딩된 수치 발견 (이동속도 200)

<제안>
- 없음

---

## 4. Artist (아트 에이전트)

### 역할
- 도형 비주얼 일관성 유지
- 색상 팔레트 관리
- 에셋 규격 정의

### 규칙
- MVP는 Phaser.js 도형만 사용 (외부 이미지 금지)
- 색상은 반드시 팔레트에서만 사용
- 모든 도형 크기는 config에서 관리

### 색상 팔레트
배경:       #0a0a0f (거의 검정)
격자선:     #1a1a2e (어두운 남색)
플레이어:   #ffffff (흰색)
적:         #ff4444 (붉은색)
보스:       #ff0000 (강한 붉은색)
아이템:     #00ffcc (청록색)
UI:         #ffffff (흰색)
경험치:     #ffd700 (금색)

---

## 5. Tester (테스트 에이전트)

### 역할
- 게임 밸런스 검토
- 버그 재현 및 보고
- 플레이어 경험 평가

### 규칙
- 실제로 플레이하는 관점에서 평가
- 버그 보고 시 재현 방법 명시
- 밸런스 수치 변경은 Planner에게 요청

### 출력 형식 
< 테스트 결과 > 

테스트 항목: 웨이브 1 클리어
결과: 통과 / 실패
버그: 적이 화면 밖에서 스폰됨
재현 방법: 게임 시작 후 30초 대기
밸런스 의견: 적 이동속도가 너무 빠름

---

## 에이전트 작업 흐름
규열 (관리자)
→ Planner에게 기능 요청
→ Planner가 Coder에게 작업 지시
→ Coder 구현 완료
→ Reviewer 리뷰
→ Tester 테스트
→ 문제 없으면 규열에게 보고
→ 문제 있으면 Coder에게 재작업 지시
